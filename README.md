# AONE GLAMPING VILLA — 予約システム デモ

グランピング施設を題材にした **予約フォーム（顧客側）＋ 予約管理画面（運用側）** の一体型デモです。
架空施設・ダミーデータによるポートフォリオ作品。デザインから実装・運用設計まで一人で担当。

🔗 **Live demo**
- 予約フォーム：https://aone-glamping-demo.vercel.app/
- 管理画面：https://aone-glamping-demo.vercel.app/admin.html

## 特徴

### 予約フォーム（お客様側 / `index.html`）
- 棟・日程・人数・オプション選択 → **リアルタイム料金計算**（人数/棟で加算ロジック分岐）
- **クライアントサイドバリデーション**（必須・本日以降の日付・人数上限・電話/メール形式）
- 確認モーダル → 完了画面の2段フロー
- レスポンシブ対応

### 管理画面（運用側 / `admin.html`）
- **KPIサマリー**：本日チェックイン / アウト / 今週の予約 / 稼働棟数（本日基準で自動計算）
- **一覧ビュー**：ステータス色分け・棟/状態/検索の絞り込み・行クリックで詳細ドロワー・状態変更・CSV出力
- **カレンダービュー**：月グリッドで宿泊状況と空室（稼働 n/5）を可視化

## 設計の考え方
「きれいな画面」ではなく **運用が楽になる画面**（対応漏れを防ぐ・探す手間を減らす）を起点に設計。
配色・タイポ・余白・コンポーネントは [`DESIGN-SYSTEM.md`](./DESIGN-SYSTEM.md) にトークンとして定義し、フォーム/管理画面で共通化。

## 技術
HTML / CSS / Vanilla JavaScript（フレームワーク非依存・単一ファイルで動作）

## 備考
架空施設のデモです。データはすべてダミーで、フォーム送信・状態変更は保存されません。
実運用では送信先（メール / スプレッドシート / Supabase 等）への接続を追加します。
